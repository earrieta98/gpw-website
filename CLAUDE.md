# GPW — Website Project

## Project Overview
Building the website for Global Precision Works (GPW), a B2B manufacturing company in Monterrey, Mexico. Two divisions under one domain (gpw-solutions.com):

## Company Naming Rules (MANDATORY)
- **Legal name:** Global Precision Works
- **Short name / acronym:** GPW
- **Domain:** gpw-solutions.com (domain only — NOT the company name)
- **NEVER use "GPW Solutions" as the company name** — this is incorrect. The domain contains "solutions" for availability reasons only.
- In copy: Use "Global Precision Works (GPW)" on first mention per page, then "GPW" thereafter.
- In title tags / meta: Use "GPW" (short, fits character limits)
- In legal / formal contexts: Use "Global Precision Works"
1. **Contract Manufacturing** — CNC precision machining (existing business, placeholder on site)
2. **Electromechanical Assembly (EMS)** — Multi-vertical assembly services (primary focus of this project)

## Target Audience
U.S. OEM procurement/supply chain decision-makers looking to nearshore electromechanical assembly to Mexico.

## Industries Served (ALL must have equal representation on the site)
- AI server rack assembly (hyperscaler & enterprise OEMs)
- Industrial equipment assembly (controls, light machinery, sub-assemblies)
- Telecom hardware (cabinets, modules, system integration)
- Medical device/equipment assembly (controlled specs)
- Automotive sub-assemblies (fixtures, integration — not heavy Tier 1)
- Energy sector (equipment cabinets, system integration)
- Aerospace sub-assemblies (full traceability)
- Appliances & white goods assembly

**IMPORTANT: The site should NOT feel like an AI server company. Target balance is ~40% AI/servers, ~60% other industries. Each industry gets a dedicated page.**

## Core Services (cross-industry)
- Cable and wire harness assembly
- Box build integration (mechanical + electrical + cabling + testing)
- End-of-line functional testing, burn-in, and quality assurance
- Full system integration (mechanical structure + component integration + cable routing + firmware loading + packaging)
- Enclosure & cabinet assembly

## Brand Identity
- **Colors:** Teal #23555A (primary), Coral #ED835E (CTA/accent), Deep Blue #2A4E64 (secondary), Lime #ADCF91 (highlights)
- **Typography:** Lato (sans-serif, primary — used for headlines, body, navigation, everything). Trirong (serif) is ONLY for small visual accents — a tagline, a decorative label, a pull quote. NEVER use Trirong as the default headline font.
- **Brand manual:** `MANUAL_IDENTIDAD_GPW_2025.docx` in project root

## Content Tone
- All copy written as if GPW is already operational and actively seeking OEM clients
- Confident but not arrogant: "engineered for", "ready to scale", "built to deliver"
- No hesitation, no "coming soon" language (except for Contract Manufacturing placeholder)
- English-only content targeting U.S. audience

## Strategic Context
- GPW is seeking investors; the site must impress potential investors by showing market readiness
- The investor is NOT the target audience — they're a spectator who gets convinced by seeing GPW knows how to reach the market
- Primary conversion goal: RFQ (Request for Quote) submissions

## Notion Integration (MANDATORY)
Every time you generate a deliverable that contains actionable tasks, action items, implementation steps, or to-do items, you MUST automatically create those tasks in the GPW Notion workspace using the Notion tools. This includes but is not limited to: content to write, pages to build, configurations to set up, profiles to create, audits to run, and any checklist items with deadlines.

**Notion Structure:**
- **Root page:** GPW Solutions (ID: 31a3748a-868d-81c2-a70e-e24f950ca14e)
- **Sub-pages:** Marketing, Pagina Web, Operaciones, Inversion
- **Tasks database:** "Tareas GPW" (data_source_id: c4bfc70f-c935-4ea2-a0f9-5efb5528e2aa)

**Task fields:**
- Tarea (title), Status (To Do/In Progress/Done/Blocked), Prioridad (Alta/Media/Baja)
- Fase (Phase 0-5, Ongoing), Categoria (Marketing/Pagina Web/Operaciones/Inversion)
- Fecha Limite (date), Fuente (source document), Responsable (person/team), Notas (context)

**Rules:**
- Create tasks immediately after generating a deliverable — don't wait to be asked
- Set appropriate priority based on project phase and business impact
- Include the source document name in the "Fuente" field
- Add enough context in "Notas" so the task is actionable without re-reading the full document
- NEVER mark a task as "Done" in Notion without explicitly asking the user first. A delivered file does NOT mean the task is done — the user may have observations or revisions. Always ask: "¿Lo marcamos como terminado en Notion?" and only update to "Done" after confirmation
- When starting work on a task, update its Status to "In Progress" in Notion
- If a task is blocked, update its Status to "Blocked" and add the reason in Notas
- At the START of every session, check Notion for any tasks that should be "In Progress" or upcoming based on deadlines
